## OpenDevin: An Open Platform for AI Software Developers as Generalist Agents

**Authors:** Xingyao Wang, Boxuan Li, Yufan Song, Frank F. Xu, Xiangru Tang, Mingchen Zhuge, Jiayi Pan, Yueqi Song, Bowen Li, Jaskirat Singh, Hoang H. Tran, Fuqiang Li, Ren Ma, Mingzhang Zheng, Bill Qian, Yanjun Shao, Niklas Muennighoff, Yizhe Zhang, Binyuan Hui, Junyang Lin, Robert Brennan, Hao Peng, Heng Ji, Graham Neubig

**arXiv ID with link:** [2407.16741](https://arxiv.org/abs/2407.16741)

**Published Date:** 2024-07-23

**Abstract:** OpenDevin is an open-source platform for the development of powerful and flexible AI agents that interact with the world in similar ways to those of a human developer: by writing code, interacting with a shell, and browsing the web. The project is open source and aims to democratize the development of AI software engineers. We provide a set of benchmarks for evaluating these agents on tasks such as software engineering, web development, and data science. Our experiments show that current models can be effective in these environments, but there is significant room for improvement in agentic coding workflows.

## Agentless: Demystifying LLM-based Software Engineering Agents

**Authors:** Chunqiu Steven Xia, Yinlin Deng, Sumanth Dathathri, Yizhe Zhang, Binyuan Hui, Junyang Lin, Robert Brennan, Hao Peng, Heng Ji, Graham Neubig

**arXiv ID with link:** [2407.01489](https://arxiv.org/abs/2407.01489)

**Published Date:** 2024-07-01

**Abstract:** We present Agentless, a novel approach to software engineering using LLMs that avoids complex agent architectures. By employing a straightforward workflow for issue localization, fix planning, and patch generation, Agentless achieves high performance on benchmarks for automated code repair and feature addition. This work provides insights into effective strategies for agentic coding with large language models, showing that simplicity can often outperform more elaborate systems in resolving real-world GitHub issues.

## CodeAgent: Enhancing Code Generation with Tool-Integrated Agent Systems

**Authors:** Kechi Zhang, Ge Li, Jia Li, Zhuo Li, Zhi Jin

**arXiv ID with link:** [2405.16721](https://arxiv.org/abs/2405.16721)

**Published Date:** 2024-05-26

**Abstract:** This paper introduces CodeAgent, a tool-integrated agent system for enhancing code generation capabilities of LLMs. By allowing agents to use external tools for searching, executing, and debugging code, it enables more agentic coding processes. Experimental results on various benchmarks show improvements in code quality and correctness through iterative tool use and agent collaboration, demonstrating the value of integrated tool use in autonomous coding tasks.

## MapCoder: Multi-Agent Code Generation for Competitive Programming

**Authors:** Md. Ashraful Islam, Mohammed Eunus Ali, Md. Saiful Islam

**arXiv ID with link:** [2405.11403](https://arxiv.org/abs/2405.11403)

**Published Date:** 2024-05-18

**Abstract:** Large language models have advanced code generation, yet they face challenges with complex problems requiring iterative refinement. MapCoder introduces a multi-agent system for agentic coding where different agents handle problem analysis, code synthesis, and verification. This collaborative approach leads to better solutions for competitive programming tasks, highlighting the benefits of agentic workflows in coding and achieving higher pass rates on benchmarks like Codeforces.

## AutoCodeRover: Autonomous Program Improvement

**Authors:** Yuntong Zhang, Haifeng Ruan, Zhiyang Chen, Zhiyuan Ma, Pengfei He, Mike Papadakis, Maxime Cordy, Xiaofei Xie, Andreas Zeller, Yang Liu

**arXiv ID with link:** [2404.05427](https://arxiv.org/abs/2404.05427)

**Published Date:** 2024-04-08

**Abstract:** We propose AutoCodeRover, an autonomous agent designed for program improvement tasks. The agent navigates large codebases, identifies relevant code sections, and applies changes to fix bugs or enhance functionality using LLM-powered reasoning. This represents a step towards fully agentic coding systems that can maintain and evolve software projects independently, with evaluations showing success on real open-source repositories for bug fixing and feature addition.
