Weebot Documentation

Production-grade AI agent framework for Windows 11. 37 tools, 6 flow types, 11 agent roles.

Installation

git clone https://github.com/georgehadji/weebot.git
cd weebot
pip install -r requirements.txt
cp .env.example .env  # Add your API keys
python -m cli.main health

Configuration

Set at least one AI provider key in .env:

# Required (at least one)
OPENROUTER_API_KEY=sk-or-...
DEEPSEEK_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...

# Optional
WEEBOT_API_KEY=your-web-api-key      # Enables API auth
WEEBOT_MCP_API_KEY=your-mcp-key      # Enables MCP auth
SANDBOX_MODE=auto                     # auto|native|docker|wsl2
BASH_TIMEOUT=30                       # seconds
DAILY_AI_BUDGET=10.0                  # USD

Quick Start

# Run a task (Plan-Act autonomous agent)
python -m cli.main flow run "Build a REST API with FastAPI"

# Start the web UI backend
python -m weebot.interfaces.web.main

# Start the MCP server (for IDE integration)
python run_mcp.py

# Interactive REPL
python run.py --interactive

Tools Reference (37 tools)

Every tool is access-controlled by role and gated by 5-layer security.

Execution Tools

bash | Execute shell via PowerShell (Windows) or WSL2. 5-layer security analysis, timeout enforcement, output truncation.
Params: command (string), timeout (number), working_dir (string), use_wsl (bool)
python_execute | Run Python in isolated subprocess (256MB memory limit). Supports pandas, numpy, matplotlib.
Params: code (string), timeout (number, default 30)
powershell | PowerShell-specific execution with diagnostic shortcuts (system_info, processes, services).
Params: command (string), diagnostic (string)
computer_use | Desktop automation via pyautogui: mouse move/click/drag, keyboard type/press/key chords.
Params: action (string), coordinates (array), text (string)

Web & Search Tools

web_search | Multi-engine search (DuckDuckGo + Bing fallback) with optional cross-encoder reranking.
Params: query (string), num_results (int, default 5)
vane_search | AI-powered cited answers with focus modes: webSearch, academicSearch, redditSearch, youtubeSearch.
Params: query (string), focus (string)
advanced_browser | Full Playwright automation: 18 actions (launch, goto, click, fill, screenshot, JS eval, cookies, sessions).
Params: action (string), url (string), selector (string), value (string)
browser_navigator | AI-driven natural-language browser automation via browser-use + Playwright.
Params: task (string), url (string)
browser_inspector | Read-only page analysis: design tokens, computed CSS, asset inventory, element inspection.
Params: url (string), action (string)

File & Data Tools

file_editor | View, create, str_replace, insert, write_large. Path validation against null-bytes and workspace escape.
Params: command (enum), path (string), file_text (string), old_str (string), new_str (string)
knowledge | Persistent FTS5-indexed knowledge base: add_note, search, get_note, list_notes, delete_note.
Params: action (string), title (string), content (string), query (string)
video_ingest | YouTube transcript ingestion with token-accurate chunking via tiktoken.
Params: action (string), url (string)
ocr | Extract text from images via Tesseract OCR. Layout-aware mode available.
Params: image_base64 (string), language (string)

Orchestration Tools

dispatch_parallel_tasks | Spawn N independent sub-agents concurrently (foreman pattern). Each runs a full PlanActFlow.
Params: tasks (array of {description, tier})
workflow_orchestrator | DAG-based multi-agent workflow. Dependency-respecting parallelism.
Params: workflow (object with tasks and dependencies)
swarm | Goal-driven parallel agent orchestration: decompose, dispatch, synthesize.
Params: goal (string), max_agents (int)
debate | Multi-perspective analysis (optimist/pessimist/pragmatist) with reconciler synthesis.
Params: topic (string), perspectives (array)
mixture_of_agents | Parallel ensemble reasoning (4 frontier models) + aggregator synthesis (MoA pattern).
Params: prompt (string), models (array)
schedule | APScheduler: create/list/get/delete/pause/resume. Triggers: cron, interval, date, once.
Params: action (string), job_id (string), trigger_type (string), trigger_config (object)

Flows (6 types)

Flows are state machines that orchestrate multi-step agent behavior.

PlanActFlow production

Core autonomous agent. 6 states: Planning -> Executing -> Reviewing -> Updating -> Summarizing -> Completed. Includes plan-stuck detection, continuation detection, truth binding, and code review.

python -m cli.main flow run "your task description"
ChatFlow production

Conversational chat with 500-exchange sliding window. For interactive sessions without autonomous planning.

HyperAgentFlow production

Multi-agent orchestrator. Decomposes complex tasks into parallel sub-agent executions with synthesis.

SkillOptFlow experimental

Full optimization epoch: Rollout -> Reflect -> Merge -> Rank -> Apply -> Validate -> Accept/Reject -> Slow Update.

HarnessGenerationFlow experimental

Generate agent teams and skills from domain description. 6 team patterns: Pipeline, Fan-out/Fan-in, Expert Pool, Producer-Reviewer, Supervisor, Hierarchical Delegation.

MCP Server

Model Context Protocol server for IDE integration (Claude Desktop, Cursor, Windsurf).

# Start MCP server
python run_mcp.py          # stdio transport (Claude Desktop)
python run_mcp.py --sse    # SSE transport (Cursor/web clients)

Exposed Tools: bash, python_execute, web_search, file_editor, ping

Resources:

weebot://activity — Recent agent activity (newest-first, up to 50)
weebot://state — Current agent state snapshot
weebot://schedule — Scheduled jobs list
weebot://costs — Session cost and cascade statistics
weebot://tools — Available tools with role access info
weebot://skills — Installed skills with descriptions

Security Model

5-layer defense-in-depth for all command execution:

Layer 1: CommandSecurityAnalyzer
Multi-layer analysis: pattern matching, behavioral detection (download+execute), entropy analysis (encoded payloads), semantic validation.
Layer 2: BashGuard
Pattern-based safety: destructive ops (rm -rf /), system mutations (systemctl, mkfs), credential exposure, network risks, fork bombs. 4 risk levels: SAFE, SUSPICIOUS, DANGEROUS, BLOCKED.
Layer 3: ExecApprovalPolicy
Rule-based approval: AUTO_APPROVE (safe commands), ALWAYS_ASK (destructive), DENY (formatting, registry deletion). User confirmation required for risky ops.
Layer 4: PathValidator
Filesystem security: workspace boundary enforcement, null-byte injection detection, URL-encoded traversal, symlink resolution, allowed extension list.
Layer 5: CapabilityGate
Role-based access control: 11 roles with tiered tool access. Privileged tools (computer_use, terminate) require elevated permissions.
